## 周报结构

周报由两张表格组成，均使用 Markdown 格式输出。

---

### 第一张表：上周工作总结

表头字段（共 7 列）：

| 三只青蛙排序 | KR及其进展 | 上周重要任务（三只青蛙🐸🐸🐸及临时的） | 原期望的结果是什么 | 实际产出/进展（用简洁的一句话阐述，具体内容可写细节——简问简答） | 状态说明（黄色和红色要进入到本周工作计划） | 补充说明 |

**字段说明：**

- **三只青蛙排序**：用 🐸+数字 表示优先级，如 🐸1、🐸2、🐸3，数字越小越重要。三只青蛙是最重要的三件事，其余为临时任务（可不加青蛙图标，直接写序号）
- **KR及其进展**：对应的关键结果（Key Result）名称，如"发布系统重构"
- **上周重要任务**：本周实际做的具体任务，可以是多行列表
- **原期望的结果是什么**：任务开始前预期达成的目标
- **实际产出/进展**：实际完成情况，第一行用一句话总结，后续可列详细子项；如有时间范围和工作量，格式为【日期范围】（Xpd）（状态）
- **状态说明**：用颜色标记完成状态
  - 🟢 绿色：按计划完成
  - 🟡 黄色：部分完成或有风险，需进入下周计划
  - 🔴 红色：未完成或严重延期，必须进入下周计划
- **补充说明**：其他需要说明的内容，可留空

---

### 第二张表：本周工作计划

表头字段（共 7 列）：

| 三只青蛙排序 | KR | 本周重要任务 | 期望的结果 | ⭐难度系数（1-3星） | 核心措施（2星和3星需要写） | 补充说明 |

**字段说明：**

- **三只青蛙排序**：同上，🐸+数字
- **KR**：对应的关键结果名称
- **本周重要任务**：本周计划完成的具体任务
- **期望的结果**：本周任务完成后的预期产出
- **⭐难度系数**：1-3星，用 ⭐ 表示
  - ⭐：简单，常规任务
  - ⭐⭐：中等，有一定挑战
  - ⭐⭐⭐：困难，需要重点关注
- **核心措施**：2星和3星任务必须填写，说明如何克服难点
- **补充说明**：可留空

## 前置步骤：初始化文件

生成周报前，先运行脚本创建文件（脚本负责日期计算、目录创建、模板复制）。

根据情况选择对应场景：

---

### 场景 A：正常每周使用

```bash
bash scripts/new-report.sh <reports-dir>
```
输出 `REPORT_FILE=...` 和 `OKR_FILE=...`，读取这两个路径后继续生成周报内容。

---

### 场景 B：首次使用 / 新员工入职

若脚本提示「首次使用？检测到季度 OKR 文件不存在」，先初始化本季度：

```bash
bash scripts/new-quarter.sh <reports-dir>
```
输出 `OKR_FILE=...`，**编辑该文件填写本季度 OKR**（参考 `reference/okr-template.md`），完成后再运行场景 A。

新员工只需从入职当周开始记录，无需补录之前的周报。

---

### 场景 C：补录历史周报（忘记记录）

对每个漏掉的周，用 `--date` 指定该周任意一天：

```bash
bash scripts/new-report.sh <reports-dir> --date 2026-03-09
bash scripts/new-report.sh <reports-dir> --date 2026-03-16
bash scripts/new-report.sh <reports-dir>                     # 当前周
```

脚本自动算出每个日期对应的周路径并创建文件，再逐一粘贴对应周的流水账生成周报。

## OKR 读取规则

Skill 从脚本输出的 `OKR_FILE` 路径读取 OKR，格式为 Markdown 表格，列顺序：

`优先级 | KR 标题 | 目标 | 关键结果 | 负责人 | 截止日期 | 状态 | 进度 | 备注`

读取时只需关注：**优先级**（决定三只青蛙排序）和 **KR 标题**（填入周报 KR 列）。参考：`reference/okr-template.md`

### 信息提取规则

从用户提供的流水账中提取以下信息：

1. **读取 OKR**：从 `OKR_FILE` 读取 KR 标题和优先级
2. **识别任务**：提取具体做了什么事情，区分主要任务和临时任务
3. **匹配 KR**：将流水账中的任务与 OKR 中的 KR 匹配
4. **识别时间线**：提取日期、工期（pd = person day）、进度状态
5. **识别状态**：根据描述判断完成状态（完成/进行中/未完成/有风险）

### 缺失信息处理

**以下字段需要用户手动填写**，Skill 无法推断：

- **原期望的结果**（上周工作总结）：任务开始前的预期目标
- **期望的结果**（本周工作计划）：本周任务完成后的预期产出
- **核心措施**（本周工作计划，仅 ⭐⭐/⭐⭐⭐ 时）：如何克服难点

Skill 会在输出周报后，列出所有【需你填写】的字段，提示你补充。

## 输出格式

严格使用 Markdown 表格，两张表之间用标题分隔：

```markdown
## 上周工作总结

| 三只青蛙排序 | KR及其进展 | 上周重要任务（三只青蛙🐸🐸🐸及临时的） | 原期望的结果是什么 | 实际产出/进展 | 状态说明 | 补充说明 |
|:---:|:---|:---|:---|:---|:---:|:---|
| 🐸1 | KR名称 | 任务描述 | 期望结果 | 实际产出 | 🟢 | |

## 本周工作计划

| 三只青蛙排序 | KR | 本周重要任务 | 期望的结果 | ⭐难度系数 | 核心措施 | 补充说明 |
|:---:|:---|:---|:---|:---:|:---|:---|
| 🐸1 | KR名称 | 任务描述 | 期望结果 | ⭐⭐ | 核心措施 | |
```

**多行内容处理**：Markdown 表格不支持换行，使用 `<br>` 标签分隔多行内容：
```
1.服务权限控制<br>2.服务发布关键节点飞书通知<br>3.飞书用户信息同步
```

## 注意事项

- **OKR 文件必须存在**：Skill 依赖 `OKR_FILE` 来获取 KR 和优先级，先运行 `scripts/new-quarter.sh` 创建
- **OKR 格式必须是表格**：参考 `reference/okr-template.md`，KR 标题必须唯一
- **表格列宽**：Markdown 中无法精确控制，保持内容简洁
- **状态说明列**：只写颜色 emoji（🟢/🟡/🔴），不写文字，保持列宽紧凑
- **下周计划推断**：如果用户没有提供下周计划，根据上周未完成项（🟡/🔴）和合理推断填写，并注明"（待确认）"
- **青蛙数量**：通常不超过 3 个，其余任务不加青蛙图标
- **缺失字段提示**：输出完成后，列出所有【需你填写】的字段，请用户确认或补充
