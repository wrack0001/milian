## ⛔ 禁止事项

1. **禁止自行实现日期计算、目录创建、模板复制** — 必须运行脚本，脚本已处理所有边界情况（跨月、跨年、跨季度）
2. **禁止猜测文件路径** — 所有路径从脚本输出的 `REPORT_FILE=...`、`OKR_FILE=...`、`PREV_REPORT_FILE=...` 获取
3. **禁止读取 `reference/` 目录中的文件** — 模板由脚本内部使用，唯一例外：年度复盘场景需读取 `annual-review-template.md`

## 工作流程

按以下步骤顺序执行，不可跳过。

### Step 1: 判断意图，运行对应脚本

根据用户的话确定场景，执行对应命令。`<reports-dir>` 由用户提供（如 `~/weekly-reports`）。若用户未指定，先询问：「请提供周报存放目录路径」。

> 脚本路径相对于 skill 安装目录（通常为工作区根目录下的 `skills/weekly-report/`）。脚本依赖 `python3`。

| 用户意图 | 执行命令 |
|:---|:---|
| 初始化季度 / 生成周报目录 | `bash skills/weekly-report/scripts/new-quarter.sh <reports-dir>` |
| 写本周周报 / 整理工作 | `bash skills/weekly-report/scripts/new-report.sh <reports-dir>` |
| 补录历史周报 | `bash skills/weekly-report/scripts/new-report.sh <reports-dir> --date YYYY-MM-DD` |
| 年度复盘 | 无需脚本，跳到「特殊场景 — 年度复盘」 |

脚本输出中提取三个路径：
```
REPORT_FILE=/path/to/report.md        ← 周报写入位置
OKR_FILE=/path/to/okr.md              ← OKR 读取位置
PREV_REPORT_FILE=/path/to/prev.md     ← 上周周报（空值表示不存在）
```

> 若脚本提示 OKR 文件不存在，先运行 `new-quarter.sh` 初始化季度，用户编辑 OKR 后再重新运行。

**仅「初始化季度」场景**：脚本执行完毕后提示用户编辑 `OKR_FILE` 填写本季度 OKR，流程结束。

### Step 2: 读取 OKR

从 `OKR_FILE` 读取 Markdown 表格，提取两列：

- **优先级**（🐸1 / 🐸2 / 🐸3）→ 决定三只青蛙排序
- **KR 标题** → 填入周报的 KR 列

表格列顺序：`优先级 | KR 标题 | 目标 | 关键结果 | 负责人 | 截止日期 | 状态 | 进度 | 备注`

### Step 3: 收集用户输入 + 读取上周周报

- 如果用户已粘贴流水账 → 继续 Step 4
- 如果用户只说"帮我写周报"但没有粘贴 → 提示：「请粘贴本周工作记录，我来帮你整理成周报」，等待输入

同时检查脚本输出的 `PREV_REPORT_FILE`：如果非空，读取该文件的「本周工作计划」表，将 🟡/🔴 状态的任务自动带入本周，标注`（待确认）`。如果为空，跳过。

### Step 4: 解析流水账，生成周报

从用户的流水账中提取信息，按以下规则生成两张表格：

**信息提取：**
1. 识别具体任务，区分主要任务和临时任务
2. 将任务与 OKR 中的 KR 标题匹配
3. 无法匹配的任务视为「临时任务」，排在三只青蛙之后，不加青蛙图标，序号从 4 开始
4. 提取日期范围、工期（pd = person day）、进度状态
5. 根据描述判断完成状态（完成 / 进行中 / 未完成 / 有风险）

**缺失信息处理：**
- 无法推断的字段 → 填 `（待补充）`
- 可推断但不确定 → 填 `（待确认）`
- 用户未提供下周计划 → 从上周 🟡/🔴 任务推断，标注`（待确认）`

需用户手动填写的字段（始终标 `（待补充）`）：
- **原期望的结果**（上周工作总结）
- **期望的结果**（本周工作计划）
- **核心措施**（本周工作计划，仅 ⭐⭐ / ⭐⭐⭐ 时）

### Step 5: 写入文件，提示待补充项

1. 读取 `REPORT_FILE`，如果已有内容则提示用户确认是否覆盖
2. 将生成的周报写入 `REPORT_FILE`
3. 列出所有 `（待补充）` 和 `（待确认）` 的字段，提示用户补充

## 周报格式规范

周报由两张 Markdown 表格组成，多行内容用 `<br>` 分隔。

### 上周工作总结（7 列）

| 三只青蛙排序 | KR及其进展 | 上周重要任务（三只青蛙🐸🐸🐸及临时的） | 原期望的结果是什么 | 实际产出/进展 | 状态说明 | 补充说明 |
|:---:|:---|:---|:---|:---|:---:|:---|

字段说明：
- **三只青蛙排序**：🐸1 > 🐸2 > 🐸3，最多 3 个；临时任务不加青蛙图标
- **实际产出/进展**：第一行一句话总结，下方可列子项；有时间和工期时格式为 `【日期范围】（Xpd）（状态）`
- **状态说明**：🟢 完成 / 🟡 部分完成或有风险 / 🔴 未完成，只写 emoji 不写文字

### 本周工作计划（7 列）

| 三只青蛙排序 | KR | 本周重要任务 | 期望的结果 | ⭐难度系数 | 核心措施 | 补充说明 |
|:---:|:---|:---|:---|:---:|:---|:---|

字段说明：
- **⭐难度系数**：⭐ 简单 / ⭐⭐ 中等 / ⭐⭐⭐ 困难
- **核心措施**：⭐⭐ 和 ⭐⭐⭐ 必填，说明如何克服难点

## 特殊场景

### 年度复盘

当用户说"帮我做年度复盘"、"生成年度总结"时：
1. 读取该年全部季度的 `okr.md` 和所有周的 `report.md`
2. 读取 `skills/weekly-report/reference/annual-review-template.md` 获取模板格式
3. 按模板生成年度复盘，写入 `<reports-dir>/{year}/annual-review.md`
