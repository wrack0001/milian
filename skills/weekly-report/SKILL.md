## 周报结构

周报由两张 Markdown 表格组成。

### 第一张表：上周工作总结（7 列）

字段说明（仅列需要额外说明的字段，其余列名自解释）：

- **三只青蛙排序**：🐸1 > 🐸2 > 🐸3，最多 3 个；其余为临时任务，不加青蛙图标
- **实际产出/进展**：第一行一句话总结，下方可列子项；有时间和工期时格式为 `【日期范围】（Xpd）（状态）`
- **状态说明**：🟢 完成 / 🟡 部分完成或有风险（需进入下周计划） / 🔴 未完成（必须进入下周计划），只写 emoji 不写文字

### 第二张表：本周工作计划（7 列）

- **⭐难度系数**：⭐ 简单 / ⭐⭐ 中等 / ⭐⭐⭐ 困难
- **核心措施**：⭐⭐ 和 ⭐⭐⭐ 必填，说明如何克服难点

多行内容用 `<br>` 分隔。

## 前置步骤

生成周报前运行脚本获取文件路径：

```bash
bash scripts/new-report.sh <reports-dir>              # 当前周
bash scripts/new-report.sh <reports-dir> --date YYYY-MM-DD  # 补录指定周
```

脚本输出 `REPORT_FILE=...` 和 `OKR_FILE=...`，读取这两个路径后继续。

若提示 OKR 文件不存在，先运行 `bash scripts/new-quarter.sh <reports-dir>` 初始化季度，编辑 OKR 后重新运行。

## OKR 读取规则

Skill 从脚本输出的 `OKR_FILE` 路径读取 OKR，格式为 Markdown 表格，列顺序：

`优先级 | KR 标题 | 目标 | 关键结果 | 负责人 | 截止日期 | 状态 | 进度 | 备注`

读取时只需关注：**优先级**（决定三只青蛙排序）和 **KR 标题**（填入周报 KR 列）。参考：`reference/okr-template.md`

### 信息提取规则

从用户提供的流水账中提取以下信息：

1. **读取 OKR**：从 `OKR_FILE` 读取 KR 标题和优先级
2. **识别任务**：提取具体做了什么事情，区分主要任务和临时任务
3. **匹配 KR**：将流水账中的任务与 OKR 中的 KR 匹配
4. **处理未匹配任务**：无法匹配到任何 KR 的任务视为"临时任务"，排在三只青蛙之后，不加青蛙图标，序号从 4 开始递增
5. **识别时间线**：提取日期、工期（pd = person day）、进度状态
6. **识别状态**：根据描述判断完成状态（完成/进行中/未完成/有风险）

### 缺失信息与占位符

Skill 无法推断的字段填入 `（待补充）`，推断但不确定的字段填入 `（待确认）`。输出完成后列出所有占位项提示用户。

需用户手动填写（`（待补充）`）的字段：
- **原期望的结果**（上周工作总结）
- **期望的结果**（本周工作计划）
- **核心措施**（本周工作计划，仅 ⭐⭐/⭐⭐⭐ 时）

## 输出目标

生成的周报内容**写入 `REPORT_FILE` 路径**（由 `scripts/new-report.sh` 输出）。写入前读取该文件，如果已有内容则提示用户确认是否覆盖。

## 注意事项

- **下周计划推断**：用户没有提供下周计划时，根据上周未完成项（🟡/🔴）推断填写，标注`（待确认）`
- **输出完成后**：列出所有`（待补充）`和`（待确认）`的字段，提示用户补充

## 特殊场景处理

### 用户未提供流水账

当用户说"帮我写周报"但没有粘贴流水账时：
1. 读取 `REPORT_FILE` 和 `OKR_FILE`
2. 提示用户粘贴本周工作记录，等待用户输入后再生成周报

### 上周周报续接

生成周报前，检查上一周的 `report.md` 是否存在。如果存在，读取其「本周工作计划」表：
- 将上周计划中的任务作为本周「上周重要任务」的参考
- 上周状态为 🟡/🔴 的任务自动带入本周计划，标注`（待确认）`

### 年度复盘

当用户说"帮我做年度复盘"、"生成年度总结"时：
1. 读取该年全部季度的 `okr.md` 和所有周的 `report.md`
2. 按 `reference/annual-review-template.md` 模板生成年度复盘
3. 写入 `{year}/annual-review.md`
