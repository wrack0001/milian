## 字段填写指南

### 上周工作总结

#### 三只青蛙排序
- **来源**：自动从 `{year}/Q{quarter}/okr.md` 读取
- **格式**：🐸1 / 🐸2 / 🐸3
- **说明**：表示该 KR 的优先级

#### KR及其进展
- **来源**：自动从 `{year}/Q{quarter}/okr.md` 读取
- **格式**：KR 标题
- **说明**：对应的关键结果名称

#### 上周重要任务
- **来源**：Skill 从你的流水账提取
- **格式**：多行列表，用 `<br>` 分隔
- **说明**：本周实际做的具体任务

#### 原期望的结果是什么
- **来源**：**需你手动填写** ⚠️
- **格式**：一句话描述
- **说明**：任务开始前的预期目标（Skill 无法推断，必须由你提供）
- **示例**：`完成权限控制模块开发`

#### 实际产出/进展
- **来源**：Skill 从你的流水账提取
- **格式**：第一行一句话总结，后续可列详细子项；如有时间范围和工期，格式为 `【日期范围】（Xpd）（状态）`
- **说明**：实际完成情况
- **示例**：
  ```
  完成权限控制和通知模块开发
  1.服务权限控制
  2.服务发布关键节点飞书通知
  3.飞书用户信息同步
  【3.18～3.24】（5pd）（自测中）
  ```

#### 状态说明
- **来源**：Skill 推断，你可以确认或修改
- **格式**：🟢 / 🟡 / 🔴
- **说明**：
  - 🟢 绿色：按计划完成
  - 🟡 黄色：部分完成或有风险，需进入下周计划
  - 🔴 红色：未完成或严重延期，必须进入下周计划

#### 补充说明
- **来源**：可选，你主动提供
- **格式**：自由文本
- **说明**：其他需要说明的内容

### 本周工作计划

#### 三只青蛙排序
- **来源**：自动从 `{year}/Q{quarter}/okr.md` 读取
- **格式**：🐸1 / 🐸2 / 🐸3

#### KR
- **来源**：自动从 `{year}/Q{quarter}/okr.md` 读取
- **格式**：KR 标题

#### 本周重要任务
- **来源**：用户提供，或从上周未完成项推断
- **格式**：多行列表，用 `<br>` 分隔
- **说明**：本周计划完成的具体任务

#### 期望的结果
- **来源**：**需你手动填写** ⚠️
- **格式**：一句话描述
- **说明**：本周任务完成后的预期产出（这是你对下周的承诺）
- **示例**：`完成权限控制模块的单元测试和集成测试`

#### ⭐难度系数
- **来源**：Skill 推断，你可以确认或修改
- **格式**：⭐ / ⭐⭐ / ⭐⭐⭐
- **说明**：
  - ⭐：简单，常规任务
  - ⭐⭐：中等，有一定挑战
  - ⭐⭐⭐：困难，需要重点关注

#### 核心措施
- **来源**：**需你填写**（仅当难度系数为 ⭐⭐ 或 ⭐⭐⭐ 时）
- **格式**：多行列表，用 `<br>` 分隔
- **说明**：如何克服难点、关键的实施步骤
- **示例**：
  ```
  1.复用已有的权限校验框架
  2.参考 team-stability 的测试用例
  3.预留 2 天用于性能优化
  ```

#### 补充说明
- **来源**：可选，你主动提供
- **格式**：自由文本
- **说明**：其他需要说明的内容

## 使用流程

### 1. 准备 OKR 文件

```bash
# 季度初，复制模板到你的仓库
cp skills/weekly-report/reference/okr-template.md 2026/Q1/okr.md

# 编辑 okr.md，填入你的 OKR
# 提交 Git
git add 2026/Q1/okr.md
git commit -m "Q1 2026 OKR 定义"
```

### 2. 生成周报

```bash
# 每周，粘贴你的流水账，调用 weekly-report skill
# Skill 会自动：
# 1. 读取 okr.md，获取 KR 和优先级
# 2. 解析你的流水账，提取任务和进度
# 3. 生成周报草稿
# 4. 列出所有【需你填写】的字段
```

### 3. 填写缺失字段

Skill 输出后，你需要填写：
- ✏️ **原期望的结果**（上周工作总结）
- ✏️ **期望的结果**（本周工作计划）
- ✏️ **核心措施**（本周工作计划，仅 ⭐⭐/⭐⭐⭐ 时）

### 4. 保存周报

```bash
# 保存到指定路径
# 路径格式：{year}/Q{quarter}/{month}/{week}/report.md
# 例如：2026/Q1/03/03.24-03.30/report.md

git add 2026/Q1/03/03.24-03.30/report.md
git commit -m "周报：2026-03-24 ~ 2026-03-30"
```

## 常见问题

**Q: 如果某个 KR 在流水账中没有提到怎么办？**  
A: Skill 会在本周工作计划中为该 KR 创建一行，任务和期望结果留空，标注【待填写】。

**Q: 如果流水账中提到的 KR 不在 okr.md 中怎么办？**  
A: Skill 会标记为 `❓ 未知 KR`，提示你确认是否需要添加到 OKR 中。

**Q: 状态说明（🟢/🟡/🔴）是 Skill 自动判断吗？**  
A: Skill 会根据"完成度"推断，但你可以手动修改。如果任务完成度 > 80% 标记为 🟢，50%-80% 标记为 🟡，< 50% 标记为 🔴。

**Q: 难度系数怎么评估？**  
A: 根据任务的复杂度和工作量：
- ⭐：已有成熟方案，预计 1-2 天完成
- ⭐⭐：有一定技术挑战，预计 3-5 天完成
- ⭐⭐⭐：高难度，需要创新或深度研究，预计 > 5 天完成

**Q: 可以不填写某些字段吗？**  
A: 不建议。【需你填写】的字段是周报的核心，不填会影响周报的完整性和管理层的理解。
